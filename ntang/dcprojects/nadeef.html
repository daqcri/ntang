Abstract

Being a commodity data cleaning system, NADEEF aims to be extensible, generic and easy-to-deploy.

Most existing data cleaning methods and systems, either in industry or academia, employ different types of data quality rules in isolation, each time a new data cleaning problem arises or a new type of rules is considered, practitioners either build a new system from scratch or go through a painful process of customizing an existing tool. The NADEEF team designed a new data cleaning system that distinguishes between a programming interface and a core to achieve generality and extensibility.

Overview

The figure below depicts of the architecture of NADEEF. It contains three components:

  • the Rule Collector gathers user-specified quality rules;
  • the Core component uses a rule compiler to compile heterogeneous rules into homogeneous constructs that allow the development of default holistic data cleaning algorithms;
  • the Metadata management and Data quality dashboard modules are concerned with maintaining and querying various metadata for data errors and their possible fixes. The dashboard allows domain experts and users to easily interact with the system.

    About

    NADEEF is an open source project created and developed by Qatar Computing Research Institute. The project was started since 2011 that Nan Tang is handled as the PI, with Ihab Ilyas and Mourad Ouzzani as co-PIs and three interns Amr Ebaid, Ahmed Eldawy, and Michele Dallachiesa. Later Jorge Quiane-Ruiz joined for its feature of entity resolution, and Si Yin joined and led the development of NADEEF. The distributed NADEEF version, as knowns as BigDansing, is led by Jorge.
